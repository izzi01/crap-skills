# Landing Page Copy

This skill creates landing-page and homepage copy with stronger conversion logic and clearer message hierarchy.

It is based on external baseline patterns from:
- `onewave-ai/claude-skills@landing-page-copywriter`
- `guia-matthieu/clawfu-skills@conversion-copywriting`

This version keeps the useful framework structure from those sources, but removes empty conversion theater and pushes harder on audience truth, proof, specificity, and believable CTA posture.

## What this skill is for

Use it when the user wants to:
- write landing page copy from scratch
- rewrite weak homepage copy
- improve a hero, CTA, FAQ, or proof section
- make a page clearer and more persuasive
- reduce generic language and increase message specificity
- organize a page around stronger copy hierarchy

## Core rule

Do not invent persuasive energy out of thin air.

Good landing-page copy is not:
- louder adjectives
- more hype
- more urgency theater
- more startup filler

Good landing-page copy is:
- clearer about who it is for
- clearer about what problem exists
- clearer about how the offer works
- stronger on proof
- more specific about the next step

## Workflow

### 1. Capture the essentials
Before writing, identify:
- what the offer is
- who it is for
- what problem or decision it helps with
- what makes it different or better
- what proof exists
- what action the page should drive

If key facts are missing, say what is missing instead of bluffing.

### 2. Find the reader's angle
Decide:
- what the reader already knows
- what they are unsure about
- what fear, friction, or desire matters most
- what must be understood in the first screen

### 3. Choose a structure
You may use PAS, AIDA, StoryBrand, or a simpler message hierarchy.

Choose the structure that best fits the offer. Do not force a famous framework just because it is familiar.

### 4. Write the page around message hierarchy
Prefer this order unless the page clearly needs a different one:
1. audience + offer clarity
2. problem / stakes
3. solution / mechanism
4. proof
5. how it works
6. objections / FAQ
7. CTA

### 5. Push for specificity
When possible, replace abstraction with:
- named audience
- named workflow or process
- named deliverables
- concrete proof
- realistic CTA language

## What to avoid

Cut or rewrite:
- vague value propositions
- consultant/meta language
- hype without evidence
- empty transformation promises
- generic CTA text like `Submit` or `Learn more` when a clearer action is possible
- feature lists with no reader value
- fake urgency if no real deadline or scarcity exists

## Copy principles

### Audience first
The reader should feel named, not merely targeted.

### Mechanism beats puffery
If the page promises an outcome, explain how the offer gets them there.

### Proof beats polish
If a claim is strong, support it with evidence, examples, numbers, process detail, or trust surfaces.

### CTA posture matters
The CTA should match the actual commitment:
- low-friction: `See how it works`
- medium-friction: `Get your personalized plan`
- high-friction: `Book a strategy call`

Do not oversell a small step or undersell a serious one.
