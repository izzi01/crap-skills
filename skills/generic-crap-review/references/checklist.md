# Generic Crap Review Checklist

Use this checklist when reviewing homepage or landing-page copy.

## Audience
- Does the page explicitly name the real audience?
- Does it name secondary stakeholders if they matter?
- Does it reflect a real situation the audience is in?

## Problem
- Does it name a real confusion, fear, or stuck moment?
- Does it avoid abstract motivation-only language?

## Mechanism
- Can the user tell how the service works?
- Are there steps, milestones, or a visible process?

## Proof
- Are there concrete proof surfaces?
- Are claims matched by evidence?
- If evidence is weak, should the claim be softened?

## Language quality
- Remove banned filler and puffery.
- Remove internal concept language.
- Replace presentation-talk with customer-talk.

## Variant differentiation
- Does this route have 2-3 lines or signals only it would say?
- Would this still make sense if swapped onto another variant? If yes, it is probably too generic.

## CTA
- Does the CTA match the likely user behavior?
- Is the next step concrete and low-friction?
